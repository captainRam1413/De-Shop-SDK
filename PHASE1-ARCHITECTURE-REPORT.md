## 1. Architecture Overview

De-Shop SDK is an **Algorand-powered NFT marketplace SDK for games**. It provides:
- A **TypeScript SDK** (`DeShopSDK`) that wraps Algorand blockchain interactions + a Flask backend
- A **Flask backend** with AI pricing, IPFS storage, Steam auth, WebSocket events
- An **ARC-56 Algorand smart contract** (Puya ‚Üí TEAL) for on-chain escrow
- **Game plugins** for Unity, Unreal, and Minecraft
- A **Vite + React frontend** showcasing the SDK (7-tab single-page app)

### State Management

### SDK (`src/sdk/DeShopSDK.ts` ‚Äî 1,037 LOC)
Well-architected, packageable SDK:
- **Mint:** `mintSkin()` ‚Üí backend `/mint` ‚Üí creates ASA with ARC-19 metadata
- **List:** `listAsset()` ‚Üí contract `listAsset` call (escrow)
- **Buy:** `buyAsset()` ‚Üí contract `buyAsset` call (atomic payment + transfer)
- **Cancel:** `cancelListing()` ‚Üí contract `cancelListing`
- **History:** `getPlayerAssets()` ‚Üí indexer query
- **AI Pricing:** `getAIPrice()` ‚Üí backend `/ai/price`
- **Steam:** `getSteamSkins()`, `linkSteamAccount()` ‚Üí backend
- **Caching:** TTL cache (5-min default), retry with exponential backoff
- **Events:** typed emitter (mint, list, buy, cancel, transfer, error, walletChanged)

### Wallet Connection Flow
1. User clicks "Connect Wallet" ‚Üí `WalletModal.premium` opens
2. `@txnlab/use-wallet-react` WalletManager handles Pera/Defly
3. On connect, `SDKProvider` calls `sdk.setWalletSigner(activeAddress, transactionSigner)`
4. SDK now has the signer for all contract calls

### AI Pricing Engine
- **Algorithm:** Weighted Least Squares (WLS) linear regression with 7-day half-life time-decay
- **Features:** recency, rarity, demand, market sentiment (4 features)
- **Output:** suggested price in microALGO
- **Training:** requires ‚â•5 buy transactions; auto-trains on `/ai/train` call
- **Fallback:** rule-based pricing when model is untrained

### Price Oracle
- **Tier 1:** Live CoinGecko API (60s cache)
- **Tier 2:** Last-known price
- **Tier 3:** Hardcoded $0.20/ALGO

## 4. Smart Contract Structure

**Language:** Puya (Python ‚Üí TEAL v11)
**Standard:** ARC-56, ARC-22, ARC-28
**Network:** Algorand Testnet, App ID `758710979`

### 4 ABI Methods
1. **`setupAsset(asset: uint64, mbr_pay: pay)`** ‚Äî Contract opts into ASA. Requires 0.1 ALGO payment. No access control.
2. **`listAsset(axfer: axfer, price: uint64, creator: address, royalty_bps: uint64)`** ‚Äî Seller transfers 1 ASA to contract, listing stored in box `b"list_"+itob(asset_id)`.
3. **`buyAsset(asset: uint64, payment: pay)`** ‚Äî Splits payment: seller gets `price - royalty`, creator gets `royalty = price * royalty_bps / 10000`. Returns ASA to buyer.
4. **`cancelListing(asset: uint64)`** ‚Äî Seller-only, returns ASA.

### State
- **Box:** `listings` (key: `b"list_"+itob(asset_id)`, value: 80-byte `ListingRecord{seller, price, creator, royalty_bps}`)
- **No global/local state**
- **MBR:** ~2,900 ¬µALGO per listing

## 5. Game Plugins

### Unreal (C++, 2,012 LOC)

- **9/11 API routes wrong** (`/nft/mint` vs `/mint`, etc.)
- **WS URL wrong** (`/ws` vs `/socket.io/`)
- `Authenticate` hardcodes `signature = nonce`
- No Algorand SDK / wallet signing

### Minecraft (Java, 809 LOC)
- **The only plugin that works end-to-end** for read-only data
- 8 subcommands: `link/unlink/skins/equip/unequip/market/info/reload`
- Skins ‚Üí real in-game items (DIAMOND_SWORD with enchant glint)
- **Game-breaking bug:** equipping a skin gives a real DIAMOND_SWORD with SHARPNESS I ‚Üí free endgame gear
